# HLA Annual Statement Downloader

Downloads Life Insurance Premium Statements from HLA portal (portal.hla.com.my).

## Workflow

**Always ask Master KT:**
1. Which agent code? (e.g. A0022418 = KT, A0037263 = THAM MAY PENG)
2. Which life assured name(s)?
3. Which year? (default: latest/2025)

Then run the script and send the PDF(s) via WhatsApp.

## Key Portal Facts

- **Login:** ktgoo2880 / PortalKT88-2
- **Session timeout:** ~2 min — always navigate via menu, not direct URL
## Correct Navigation Path (CRITICAL — follow this order)

1. **Login** → home page
2. Go to **Servicing → Policy** menu
   - URL: `https://portal.hla.com.my/cms/Servicing/Policy-Info.aspx`
3. From the Policy page, look for **Letter/Statement** section → click **"Life Insurance Premium Statement"**
   - This establishes the session-encoded URL
4. **THEN** on the search form:
   - Set agent code FIRST (all 3 fields + fire change/blur events — see Step 4a below)
   - Type Policy Number **OR** Life Assured Name **OR** Assured Name
   - Change the year
   - Click **"Get Record"**

⚠️ **Session timeout:** ~2 min — always navigate via menu, not direct URL

**Search page URL:** `https://portal.hla.com.my/LifeInsurancePremium/WebUI/LifeInsurancePremiumSearch.aspx`
  - Only works when navigated to AFTER Policy-Info.aspx (session must be established via menu click)

- **Form field IDs:**
  - Agent dropdown: `cphContent_ddlSearchAgentCode`
  - Agent text helper: `cphContent_txtComboboxID` and `cphContent_txtSearchAgentCode` (set these too)
  - Life Assured Name: `cphContent_txtLifeAssuredName`
  - Assured Name (policy owner/payer): `cphContent_txtAssuredName`
  - Policy No: `cphContent_txtPolicyNo`
  - Year: `cphContent_cboDUYear` (values: '', '2025', '2024', '2023')
  - Search button: `cphContent_btnGetRec`

- **⚠️ CRITICAL — Setting agent code correctly (Step 4a):**
  Must set ALL THREE fields + fire change AND blur events, then wait 2s:
  ```js
  var dd = document.getElementById('cphContent_ddlSearchAgentCode');
  dd.value = 'A0037263';
  dd.dispatchEvent(new Event('change'));
  dd.dispatchEvent(new Event('blur'));
  
  var f = document.getElementById('cphContent_txtComboboxID');
  f.value = 'A0037263';
  f.dispatchEvent(new Event('input'));
  f.dispatchEvent(new Event('change'));
  f.dispatchEvent(new Event('blur'));
  
  var f2 = document.getElementById('cphContent_txtSearchAgentCode');
  f2.value = 'A0037263';
  f2.dispatchEvent(new Event('input'));
  f2.dispatchEvent(new Event('change'));
  ```
  Wait 2 seconds after this before filling other fields.
  **Verify:** Check `document.getElementById('cphContent_ddlSearchAgentCode').value` == target agent before proceeding.

- **Search tips:**
  - Search by **Life Assured Name** for the insured person
  - Search by **Assured Name** to find policies where someone is the payer (e.g. parent paying for child)
  - Use **Year = View All** (empty) to see all available years
  - Results show: policy no, life assured, IC, assured name, assured IC, year, ViewPdf link
  - Multi-page results: click page number links to navigate

- **New 2025 policies (UL202528xxxxx):** Annual statements NOT yet generated by HLA — return "No record". Check back later.

- **PDF download:** Click ViewPdf link → cookies carry over → direct HTTP download works if `%PDF` in first 100 bytes. Otherwise use CDP `Page.printToPDF`.

## Script

Use `scripts/hla_life_ins_prem_stmt.py` as the base. It:
- Starts Chrome on port 9250 (reuses existing if running)
- Logs in, navigates: **Servicing → Policy → Letter/Statement → Life Insurance Premium Statement**
- Searches by life assured name OR policy number under specified agent
- Downloads all ViewPdf links as real PDFs
- Handles multi-page results

**To run for new searches**, modify the `SEARCHES` list at the top and run:
```bash
python3 scripts/hla_life_ins_prem_stmt.py
```

## Output & Delivery

1. Save PDFs to `~/WisSend/` (auto-syncs to `~/.openclaw/workspace/media/`)
   - Name format: `{LIFE_ASSURED}_{AGENT_TAG}_{POLICY_NO}_{YEAR}.pdf`
   - Example: `WONG_YIK_CHUN_A0037263_UL202528370623_2025.pdf`
2. Wait ~3 seconds for auto-copy to complete
3. **Send PDF via WhatsApp using `openclaw message send`:**
   ```bash
   openclaw message send --channel whatsapp --target +60125226892 \
     --media ~/WisSend/{filename}.pdf \
     --message "HLA Annual Statement 2025 — {LIFE_ASSURED}"
   ```
   - For multiple PDFs, run the command for each file
   - Works for: PDFs, images, documents
4. Delete PDFs from `~/WisSend/` after sending

**⚠️ IMPORTANT:** After downloading, ALWAYS verify with `file ~/WisSend/{filename}` that it's a real PDF (not HTML). If it's HTML, the portal failed — re-login and retry.
