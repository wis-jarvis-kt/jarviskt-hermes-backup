## 🐛 Conversion Codex Debugging Session — 2026-03-31 Morning

### Bugs Found & Fixed

#### Bug 1: Vercel alias stale deployment
- Symptom: Dashboard showing old code (401 Missing authorization header)
- Cause: `--prod` flag created new deployment URL but alias `conversion-codex-dashboard.vercel.app` still pointed at old build
- Fix: Manually re-pointed alias to correct deployment

#### Bug 2: `ignoreDuplicates: true` + `.select().single()` returns null for existing rows
- Symptom: wa-webhook silently skipped all processing for existing leads (people already in DB)
- Cause: Supabase returns empty when `ignoreDuplicates: true` and row exists — `.single()` returns null
- Fix: Rewrote person lookup as fetch-first → insert-if-missing → retry-on-race pattern in `getOrCreatePerson()`
- File: `/Users/ktoclaw/.openclaw/workspace/conversion-codex/supabase/functions/wa-webhook/index.ts`
- **Lesson: Never use `ignoreDuplicates: true` with `.select().single()` — always fetch first**

#### Bug 3: `supabase.functions.invoke()` is fire-and-forget and gets killed
- Symptom: ai-respond never ran when called from within wa-webhook edge function
- Cause: Supabase edge functions terminate as soon as the outer function returns — background tasks are killed
- Also: `await fetch()` to public Supabase URL from inside an edge function hits timeout before completing
- Fix: Dashboard now calls ai-respond **directly** via its own fetch, bypassing inter-function invocation entirely
- **Lesson: Never rely on supabase.functions.invoke() for chained edge function calls — call directly or restructure**

#### Bug 4: ai-respond had no CORS handler
- Symptom: "Failed to fetch" in browser — browser preflight (OPTIONS) got 405 back, request blocked
- Cause: ai-respond only handled POST, returned 405 for OPTIONS
- Fix: Added CORS_HEADERS constant + OPTIONS preflight handler (204) to ai-respond
- File: `/Users/ktoclaw/.openclaw/workspace/conversion-codex/supabase/functions/ai-respond/index.ts`
- **Lesson: Every Supabase edge function called from browser must handle OPTIONS preflight**
