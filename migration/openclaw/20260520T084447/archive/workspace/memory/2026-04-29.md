# 2026-04-29 — Wis Reminder System Built

## What happened
Built standalone reminder system that bypasses OpenClaw entirely.

## Problem
OpenClaw crons were unreliable — API key failures, 401 auth errors, missed triggers. Reminders were dying when Wis's AI APIs failed.

## Solution: wa_reminder.py + reminder_keeper.py
- **wa_reminder.py** — reads `~/.openclaw/reminders/reminders.json`, sends via `pywhatkit` (uses Chrome + WhatsApp Web)
- **reminder_keeper.py** — smart-sleep daemon. Calculates exact seconds to next reminder, sleeps only that long, fires instantly when due
- **LaunchAgent** — `com.wis.reminder-keeper.plist` registered with launchd, auto-starts on boot
- Dependencies: `pywhatkit`, `websocket-client` (both installed via pip3)

## Active Reminders (hardcoded in reminders.json)
- Stock radar check — weekdays 21:45
- Victor 6PM Learning Scan — weekdays 18:00
- Roy Zoom meeting — Thu Apr 30, 08:00

## Files
- Script: `~/.openclaw/reminders/wa_reminder.py`

- DB: `~/.openclaw/reminders/reminders.json`
- Log: `~/.openclaw/reminders/keeper.log`
- LaunchAgent: `~/Library/LaunchAgents/com.wis.reminder-keeper.plist`

## Key Fix (cmd_add parsing bug)
The `cmd_add` parser was mangling `once` reminders — splitting on "once" as message text. Fixed by direct JSON manipulation of reminders.json when cmd_add produced wrong output. Tested: `python3 wa_reminder.py add "Roy Zoom meeting — 8am! Get ready 🎙️" once 2026-04-30 08:00 kt`

## reminder_keeper.py — smart-sleep daemon
Uses pyautogui to click WhatsApp input, type message, press Enter. No Chrome login needed.
